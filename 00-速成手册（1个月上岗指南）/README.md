# 00 - 1个月速成手册：AI系统产品经理上岗指南

> **适用对象**：有3年+技术产品经验、熟悉新能源三电系统、会基础AI工具
> **目标**：1个月后能与AI/芯片/软件研发Team、供应商进行有效博弈
> **核心理念**：抓重点、重实战、轻理论、快速建立判断力

---

## 🎯 为什么要做速成

你的背景优势：
- ✅ 三电系统经验 → 理解BMS/电机控制/功率电子（芯片应用的核心场景）
- ✅ 产品方法论 → 需求分析、竞品拆解、供应商管理
- ✅ AI工具熟练 → 不需要从Prompt工程学起

你只需要补齐：
- 🔴 **AI核心技术判断**（什么模型适合什么场景）
- 🔴 **芯片选型逻辑**（算力/功耗/成本的平衡）
- 🔴 **系统架构认知**（端到端AI系统怎么搭）
- 🔴 **与研发对话的能力**（能听懂他们在说什么、能challenge他们）

## 📅 1个月冲刺计划（每天3-4小时）

### Week 1：AI核心技术速成（建立判断力）

**目标**：理解主流AI模型能力边界，知道什么场景用什么模型

| 天数 | 主题 | 学习重点 | 产出物 |
|------|------|----------|--------|
| D1 | **Transformer速成** | 为什么Transformer统治NLP→CV→多模态；Attention机制的本质；序列并行化 | 一张图解释Transformer vs CNN vs RNN |
| D2 | **大模型能力图谱** | GPT-4/Claude/Gemini能力对比；推理vs生成；长上下文；多模态 | 一张表：主流大模型能力矩阵 |
| D3 | **模型部署与推理** | 量化(INT8/INT4)、蒸馏、RAG、微调(Full/LoRA)；推理延迟vs精度权衡 | 一份部署方案评估模板 |
| D4 | **多模态模型** | CLIP/ViT/LLaVA；视觉+语言融合；3D理解；视频理解 | 一张图：多模态模型演进路线 |
| D5 | **具身智能速成** | RT-1/RT-2/RT-X；VLA模型；端到端vs模块化；Sim2Real | 与LinuxHermes知识对齐 |
| D6-7 | **Week1复盘+输出** | 写一份《AI模型选型速查手册》 | 1份内部参考文档 |

4. [RT-2: Vision-Language-Action Models](https://arxiv.org/abs/2307.15818) — 理解具身智能

## 🎮 与研发Team博弈的核心武器

### 武器1：技术可行性评估表

| 评估维度 | 关键问题 | 你的判断依据 |
|----------|----------|--------------|
| 模型选型 | 为什么不用更小/更快的模型？ | 精度-延迟-成本三角 |
| 芯片选型 | 为什么选这款芯片？算力利用率多少？ | Roofline模型、实际Benchmark |
| 数据方案 | 训练数据从哪来？质量如何保证？ | 数据Pipeline、标注成本 |
| 部署方案 | 端侧还是云端？延迟要求多少？ | SLA定义、网络条件 |
| 成本控制 | 单用户算力成本多少？ | TCO模型、规模化成本 |
| 迭代周期 | 模型更新频率？版本兼容性？ | MLOps流程、A/B测试 |

### 武器2：供应商谈判Checklist

```
□ 技术能力验证
  □ 是否有成功案例？
  □ 能否提供POC？
  □ 技术团队背景？
  
□ 交付能力评估
  □ 交付周期？
  □ 里程碑定义？
  □ 验收标准？
  
□ 商务条款
  □ 定价模型（一次性/License/按量）？
  □ 知识产权归属？
  □ 技术支持SLA？
  □ 退出机制？
  
□ 风险评估
  □ 供应商存续风险？
  □ 技术锁定风险？
  □ 供应链风险？
```

### 武器3：技术评审会发言框架

```
1. 需求对齐（5分钟）
   "我们的业务目标是____，技术方案需要支持____"

2. 方案理解（10分钟）
   "我理解你们的方案是____，采用____技术路线"

3. 关键质疑（15分钟）
   - "这个方案的瓶颈在哪里？"
   - "如果____场景发生，系统怎么应对？"
   - "相比竞品方案____，我们的优势是什么？"
   - "成本模型怎么算？规模化后单位成本多少？"

4. 决策推进（10分钟）
   "基于以上讨论，我建议____，下一步需要____"
```

## ✅ 上岗验收标准

1个月后，你应该能：

- [ ] **技术对话**：能参与AI/芯片技术评审，提出有深度的问题
- [ ] **选型判断**：能在30分钟内评估一个AI方案的技术可行性
- [ ] **供应商管理**：能独立进行技术供应商的评估和谈判
- [ ] **方案设计**：能设计一个端到端AI产品的技术方案框架
- [ ] **风险识别**：能识别AI项目的技术风险和交付风险

---

## 🚀 上岗后立即执行的5件事

1. **建立技术雷达**：每周跟踪3-5个AI/芯片技术动态
2. **构建供应商库**：分类整理10-20家核心供应商
3. **制作内部培训**：给团队讲一堂《AI产品经理技术速成》
4. **跑一个POC**：亲手参与一个AI项目的POC，从0到1
5. **建立度量体系**：定义AI产品的核心KPI（准确率/延迟/成本/用户满意度）

---

> ⚡ **记住**：你的目标不是成为AI研究员或芯片工程师，而是成为**能和技术团队平等对话、能做技术判断的产品经理**。
>
> 1个月后，当你坐在技术评审会上，你应该能问出让工程师停下来思考的好问题。
