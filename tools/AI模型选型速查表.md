```
├── 是 → 用GPT-4/Claude 3（云端API）
└── 否 → 预算限制？

```

## 四、模型评估指标

### 性能指标

| 指标 | 说明 | 适用场景 |
|------|------|----------|
| Accuracy | 准确率 | 分类任务 |
| F1 Score | 精确率和召回率调和 | 不平衡数据 |
| mAP | 平均精度均值 | 目标检测 |
| BLEU | 文本相似度 | 文本生成 |
| Perplexity | 困惑度 | 语言模型 |
| Latency (P50/P95/P99) | 延迟分布 | 实时系统 |
| Throughput | 吞吐量 | 高并发系统 |

### 成本指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| $/1K tokens | 每千token成本 | API定价 |
| $/inference | 每次推理成本 | 算力成本 |
| $/user/month | 每用户月成本 | 总成本/用户数 |
| TCO | 总拥有成本 | 硬件+软件+运维 |

## 五、常见误区

### ❌ 误区1：模型越大越好
**真相**：大模型成本高、延迟大。很多任务用小模型就够了。

### ❌ 误区2：精度越高越好
**真相**：精度提升1%可能需要成本翻倍。要找到精度-成本的甜蜜点。

### ❌ 误区3：一个模型做所有事
**真相**：多模型组合（MoE/Ensemble）往往比单模型更好。

### ❌ 误区4：只关注训练，不关注推理
**真相**：推理成本通常是训练成本的10-100倍。

### ❌ 误区5：忽视数据质量
**真相**：数据质量对效果的影响 > 模型选择。

## 六、快速决策表

| 你的情况 | 推荐方案 |
|----------|----------|
| 需要快速上线，预算充足 | GPT-4 API |
| 需要快速上线，预算有限 | GPT-3.5 API |
| 数据敏感，不能出域 | 私有化部署Llama 2/Qwen |
| 需要边缘部署，低延迟 | TensorRT + 量化模型 |
| 车载场景，功能安全 | 车规级芯片 + 认证模型 |
| 高并发，成本敏感 | 自研模型 + 推理优化 |
| 需要多模态 | GPT-4V / LLaVA |
| 需要代码能力 | GPT-4 / CodeLlama |
