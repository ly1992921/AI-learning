# 技术评审Checklist

> 用于AI/芯片技术方案评审，确保不遗漏关键问题

## 一、模型与算法评审

### 模型选型
- [ ] 为什么选择这个模型？有没有更好的替代方案？
- [ ] 模型精度（Accuracy/F1/mAP）能否满足业务要求？
- [ ] 模型大小和参数量是否适合目标部署环境？
- [ ] 推理延迟（Latency）是否满足SLA？
- [ ] 是否考虑过更小/更快的模型（Distill/Prune/Quantize）？

### 数据方案
- [ ] 训练数据来源？是否合法合规？
- [ ] 数据量是否足够？数据质量如何保证？
- [ ] 数据标注成本和时间？
- [ ] 是否有数据偏见（Bias）问题？
- [ ] 数据更新和迭代机制？

### 训练方案
- [ ] 训练算力需求？GPU/TPU类型和数量？
- [ ] 训练时间预估？
- [ ] 是否支持增量训练/在线学习？
- [ ] 模型版本管理和回滚机制？

## 二、芯片与硬件评审

### 芯片选型
- [ ] 目标芯片的算力（TOPS/FLOPS）是否满足模型需求？
- [ ] 实际算力利用率（Roofline分析）是多少？
- [ ] 内存带宽是否成为瓶颈？
- [ ] 功耗（W）和热设计（TDP）是否符合产品要求？
- [ ] 芯片供应链稳定性？交付周期？
- [ ] 是否有替代方案（第二供应商）？

### 部署环境
- [ ] 云端部署还是边缘部署？
- [ ] 如果是边缘：目标设备是什么？（Jetson/骁龙/地平线/自研）
- [ ] 车规级要求？（AEC-Q100、ISO 26262、ASIL等级）
- [ ] 工业级要求？（温度范围、抗干扰、可靠性）

## 三、系统与工程评审

### 架构设计
- [ ] 系统架构图是否清晰？
- [ ] 端云协同方案？数据如何流转？
- [ ] 服务拆分是否合理？微服务边界？
- [ ] 并发处理能力？QPS/TPS预估？

### 部署与运维
- [ ] 模型部署流程？CI/CD for ML？
- [ ] 模型监控指标？（精度漂移、延迟、错误率）
- [ ] A/B测试方案？
- [ ] 模型更新和回滚机制？
- [ ] 日志和可观测性？

### 安全与合规
- [ ] 数据隐私保护？（GDPR/个人信息保护法）
- [ ] 模型安全？（对抗攻击、Prompt注入）
- [ ] 可解释性要求？（医疗/金融/自动驾驶）
- [ ] 行业合规要求？

## 四、商业与交付评审

### 成本模型
- [ ] 单用户/单设备算力成本？
- [ ] 规模化后单位成本变化？
- [ ] 与竞品方案的成本对比？

### 交付计划
- [ ] POC/MVP/量产的时间节点？
- [ ] 里程碑定义和验收标准？
- [ ] 风险评估和应急预案？

### 供应商管理
- [ ] 技术供应商的资质和案例？
- [ ] 知识产权归属？
- [ ] 技术支持SLA？
- [ ] 退出机制和迁移方案？

---

## 评审会发言模板

```
1. 需求对齐（5分钟）
   "我们的业务目标是____，技术方案需要支持____"

2. 方案理解（10分钟）
   "我理解你们的方案是____，采用____技术路线"

3. 关键质疑（15分钟）
   - "这个方案的瓶颈在哪里？"
   - "如果____场景发生，系统怎么应对？"
   - "相比竞品方案____，我们的优势是什么？"
   - "成本模型怎么算？规模化后单位成本多少？"
   - "数据从哪里来？质量怎么保证？"

4. 决策推进（10分钟）
   "基于以上讨论，我建议____，下一步需要____"
```
