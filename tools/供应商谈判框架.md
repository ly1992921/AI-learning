# 供应商谈判框架

> 用于AI芯片、模型、云服务供应商的技术商务谈判

## 一、谈判前准备

### 信息收集
- [ ] 供应商背景（成立时间、融资情况、核心团队）
- [ ] 技术能力（自研vs集成、专利数量、技术路线）
- [ ] 成功案例（同行业、同场景、同规模）
- [ ] 市场口碑（客户评价、行业排名）
- [ ] 竞争对手（替代方案、竞品对比）

### 谈判目标设定
- [ ] 理想目标（Best Case）
- [ ] 可接受目标（Target）
- [ ] 底线（Walk-away Point）

---

## 二、技术能力评估

### 技术验证
```
□ POC方案
  □ POC范围定义（功能、性能、时间）
  □ 验收标准（量化指标）
  □ 双方投入资源
  □ POC失败的处理方式

□ 技术团队
  □ 核心技术人员背景
  □ 技术支持响应时间
  □ 驻场支持可能性

□ 技术文档
  □ API文档完整性
  □ SDK和开发工具
  □ 示例代码和教程
```

### 性能基准
```
□ 性能测试
  □ 测试环境（与生产环境一致性）
  □ 测试数据集（代表性）
  □ 性能指标（延迟、吞吐、精度）
  □ 对比基准（竞品或开源方案）

□ 可扩展性
  □ 并发处理能力
  □ 数据量增长的处理方式
  □ 模型更新的影响
```

---

## 三、商务条款谈判

### 定价模型
```
□ 定价方式
  □ 一次性采购（License）
  □ 订阅制（按月/按年）
  □ 按量计费（API调用次数）
  □ 混合模式

□ 价格结构
  □ 基础费用
  □ 超出部分单价
  □ 阶梯价格（量越大越便宜）
  □ 预付费折扣

□ 成本透明度
  □ 算力成本明细
  □ 存储成本明细
  □ 网络成本明细
```

### 合同条款
```
□ 知识产权
  □ 模型所有权归属
  □ 数据所有权归属
  □ 生成内容归属
  □ 专利许可范围

□ 服务等级协议（SLA）
  □ 可用性承诺（99.9%？99.99%？）
  □ 响应时间承诺
  □ 故障恢复时间（RTO）
  □ 数据恢复点（RPO）
  □ 赔偿机制

□ 数据安全
  □ 数据存储位置（国内/海外）
  □ 数据加密方式
  □ 数据访问权限
  □ 数据删除机制
  □ 合规认证（等保/ISO 27001）
```

### 退出机制
```
□ 合同期限
  □ 初始期限
  □ 续约条件
  □ 提前终止条件

□ 数据迁移
  □ 数据导出格式
  □ 迁移支持
  □ 迁移时间窗口

□ 技术锁定风险
  □ 专有技术依赖度
  □ 替代方案评估
  □ 迁移成本预估
```

## 四、风险评估

### 供应商风险
| 风险类型 | 评估问题 | 应对措施 |
|----------|----------|----------|
| 财务风险 | 供应商财务状况？是否盈利？ | 要求提供财报；分散供应商 |
| 技术风险 | 技术路线是否主流？ | 选择开放标准；避免深度定制 |
| 交付风险 | 历史交付准时率？ | 分阶段交付；里程碑付款 |
| 人员风险 | 核心团队稳定性？ | 知识转移要求；文档完整性 |
| 合规风险 | 是否符合行业监管？ | 合规审查；法律条款保护 |

### 供应链风险
- [ ] 芯片供应（地缘政治影响？）
- [ ] 模型服务（API依赖？）
- [ ] 数据服务（数据源稳定性？）

## 五、谈判策略

### 议价技巧
1. **锚定效应**：先提出理想价格，逐步向目标靠拢
2. **捆绑谈判**：将多个项目捆绑，争取整体优惠
3. **竞争压力**：暗示有替代方案，但保持诚意
4. **长期合作**：强调长期合作意愿，换取更好条件
5. **分阶段**：先小规模试点，再规模化谈判

### 谈判底线
```
绝对不能接受的：
- 数据所有权转移给供应商
- 独家绑定（无退出机制）
- 技术黑盒（无文档/无API）
- SLA低于业务要求
- 价格超过预算50%以上
```
