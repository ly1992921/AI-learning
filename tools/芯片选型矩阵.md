## 三、关键评估维度

### 1. 算力指标解读

| 指标 | 含义 | 注意点 |
|------|------|--------|
| TOPS | 每秒万亿次操作 | 通常是INT8，峰值算力 |
| TFLOPS | 每秒万亿次浮点运算 | 分FP16/FP32/FP64 |
| 有效算力 | 实际利用率 | 通常只有峰值的30-70% |
| Memory BW | 内存带宽 | AI推理往往是Memory Bound |

### 2. 功耗与散热

| 场景 | 功耗预算 | 散热方式 |
|------|----------|----------|
| 数据中心 | 300-700W/卡 | 液冷/风冷 |
| 边缘服务器 | 50-150W | 风冷 |
| 车载 | 10-65W | 风冷/液冷 |
| 机器人 | 5-60W | 被动/风冷 |
| 手机/IoT | <5W | 被动 |

### 3. 软件生态

## 五、与供应商谈判要点

### 必须问的问题

1. **算力承诺**
   - 标称算力是多少？实际利用率能达到多少？
   - 有没有Benchmark数据？（MLPerf？）

2. **软件支持**
   - 支持哪些框架？（PyTorch/TensorFlow/ONNX）
   - 模型转换工具成熟度？
   - 是否有预优化模型库？

3. **供应链**
   - 供货周期？（芯片短缺时期尤其重要）
   - 是否有替代方案？
   - 长期供货承诺？

4. **技术支持**
   - 技术支持响应时间？
   - 是否有现场支持？
   - 培训资源？

5. **知识产权**
   - 芯片IP授权模式？
   - 是否有技术锁定风险？

## 六、车载芯片特别注意事项

### 功能安全等级

| ASIL等级 | 风险程度 | 典型应用 |
|----------|----------|----------|
| ASIL-D | 最高 | 自动驾驶主控 |
| ASIL-B | 高 | ADAS |
| QM | 质量管理 | 信息娱乐 |

### 车规认证

- [ ] AEC-Q100（芯片可靠性）
- [ ] ISO 26262（功能安全）
- [ ] ASPICE（软件开发流程）
- [ ] 网络安全（ISO/SAE 21434）

### 供应链安全

- [ ] 芯片是否受出口管制？
- [ ] 是否有国产替代方案？
- [ ] 第二供应商策略？

> 💡 **核心原则**：芯片选型不是选最强的，而是选**最适合当前场景（算力/功耗/成本/生态/供应链）**的。
