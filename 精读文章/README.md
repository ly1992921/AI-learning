## 🏷️ 精读模板结构

每篇精读包含以下板块（参考LinuxHermes格式）：

```
# 精读 {编号}：{论文标题}

## 0. 一句话定位
## 1. 先给结论：这篇论文到底证明了什么
## 2. 为什么这篇论文重要
## 3. 用产品经理的背景理解它
## 4. 论文要解决的核心问题
## 5. 总体架构
## 6. 核心方法 {N} 个
## 7. 数据
## 8. 实验与结果
## 9. 论文的真正创新点
## 10. 与相关工作的关系
## 11. 局限分析（产品视角）
## 12. 落地启发
## 13. 读论文时应该抓的图/表
## 14. 产品决策启发
## 15. 记忆钩子
```
