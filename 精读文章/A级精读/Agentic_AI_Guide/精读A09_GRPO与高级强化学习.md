# 精读A09：GRPO（组相对策略优化）与高级强化学习

## 0. 一句话定位

**GRPO（Group Relative Policy Optimization，组相对策略优化）是去掉了Critic价值网络的PPO变体——用"组内平均分"替代"价值模型预测值"，在LLM强化学习训练中已成为事实上的默认算法。**

GRPO由DeepSeek在DeepSeekMath工作中首次提出，后在DeepSeek-R1中大规模验证，已替代PPO成为多数开源对齐框架（TRL、OpenRLHF、veRL）的默认RL算法。

---

## 1. GRPO的动机：为什么不要Critic了？

### PPO中Critic的三大问题

PPO（Proximal Policy Optimization）需要一个价值网络（Critic）来估计状态的价值函数 V(s)，即"给定当前部分输出，最终的奖励期望值是多少"。但在大语言模型场景下，这个Critic存在三个核心问题：

#### 问题1：内存爆炸（Memory）

- 价值头（Value Head）通常共享策略模型（Policy）的骨干网络。
- 70B参数的模型，骨干网络占约140GB显存。
- 如果Critic使用独立的模型，内存需求直接翻倍。
- 这就意味着：你训练一个70B模型，等于还要再维护一个70B规模的Critic。

#### 问题2：价值预测极度不准确（Accuracy）

- 预测"一个部分完成的句子最终能得多少分"，本质上是一个极困难的问题。
- 比如模型刚输出了"Therefore, the answer is"，你要预测它最终是否正确——这是几乎不可能准确预测的。
- 价值函数经常给出错误估值 → 导致Advantage（优势值）计算错误 → 进而导致梯度方向错误。
- 原文原话："The value function is often wrong → wrong advantages → wrong gradient direction."

#### 问题3：价值网络收敛慢（Training Instability）

- 价值头需要大量样本才能收敛。
- 在RL训练的早期阶段，价值模型给出的预测噪声极大，反而会干扰策略网络的稳定学习。

### GRPO的核心洞察：用经验分布替代价值函数

> "Instead of learning V(s), estimate it empirically from a group of samples."

GRPO的关键思想非常简洁：对同一个prompt生成多个回答（称为一个Group），计算这些回答的奖励值，然后使用**组内的统计量**（均值和标准差）替代价值函数。

## 2. PPO vs GRPO对比

| 对比维度 | PPO | GRPO |
|---------|-----|------|
| **价值模型** | 需要Critic网络，预测V(s) | 不需要Critic，用组内统计量替代 |
| **基线计算** | 用价值网络预测状态价值 | 用组内平均奖励作为基线 |
| **Advantage** | A = R - V(s)（预测值） | Â = (ri - μ_G) / σ_G（经验值） |
| **内存需求** | 高（需额外维护价值网络） | 低（省去一个完整模型） |
| **工程复杂度** | 高（双网络训练、同步） | 低（单网络，结构简洁） |
| **准确性瓶颈** | 价值网络预测精度 | 采样组内多样性 |
| **典型适用** | 传统连续控制、游戏 | LLM推理、代码生成 |

### 核心区别一句话

> **PPO需要一个"老师"（价值网络）告诉模型这个行动好不好；GRPO不需要老师，让模型在同一道题上多答几次，自己跟自己比。**

## 3. GRPO算法核心

### 3.1 算法流程（四步）

**步骤1：采样（Sample）** — 对每个prompt x，从当前策略 π_θ 中采样 G 个回答：

```
{y₁, y₂, ..., y_G} ~ π_θ(·|x)
```

**步骤2：打分（Score）** — 计算每个回答的奖励值：

```
r_i = R(x, y_i)
```

**步骤3：组内归一化（Normalize）** — 在组内对奖励做标准化：

```
Â_i = (r_i - μ_G) / σ_G

其中：
μ_G = (1/G) * Σⱼ rⱼ         （组内平均奖励）
σ_G = std({rⱼ}) + ε         （组内奖励标准差，加ε防止除零）
```

**步骤4：策略更新（Update）** — 使用PPO风格的clip机制更新策略：

```
L = E[ min( r_t(θ)·Â_i, clip(r_t(θ), 1±ε)·Â_i ) ] - β·D_KL[π_θ ‖ π_ref]
```

### 3.2 为什么组内归一化有效？

- **组平均 ≈ 价值函数**：对于同一个prompt，足够多样本的奖励平均值，可以作为价值函数V(s)的蒙特卡洛估计。
- **高于均值 = 好行动**：Â_i > 0 意味着这个回答比组内平均好，应该被加强。
- **低于均值 = 差行动**：Â_i < 0 意味着比组内差，应该被抑制。
- **除以标准差**：确保不同prompt之间的advantage尺度一致（scale-invariant），不受奖励数值范围影响。

### 3.3 组大小（G）的选择

| G | 信号质量 | 计算成本 | 建议场景 |
|---|---------|---------|---------|
| 2 | 极差（像抛硬币） | 低 | 永远不推荐 |
| 4 | 中等 | 中等 | 快速实验，简单任务（通过率>50%） |
| **8** | **良好** | **高** | **默认值，大多数任务的最佳平衡** |
| 16 | 优秀 | 很高 | 困难任务（通过率<20%） |
| 32 | 近乎完美 | 极高 | 只有算力充足且任务极难时使用 |

### 3.4 关键约束：组内必须同时有成功和失败

这是GRPO最容易踩的坑：

- **如果G个回答全部正确**（全部r=1）：所有advantage=0，完全没有学习信号！
- **如果全部错误**：同样没有学习信号。
- **黄金法则（Goldilocks Rule）**：选择通过率在20%-80%之间的prompt，且每500步重新过滤一次。

## 4. GRPO在DeepSeek-R1中的应用

### 里程碑意义

DeepSeek-R1的突破证明了GRPO的强大：

- 使用纯GRPO训练，奖励函数仅为**二元正确性奖励**（正确答案r=1，错误r=0）。
- 训练数据为数学和代码任务。
- **结果**：模型**自发涌现**了链式推理（Chain-of-Thought）、自我验证（Self-Verification）、纠错（Error Correction）等复杂推理行为——没有任何显式指令！

### 三电类比

> 这就像：你只告诉车辆"这条标定曲线是否达标（1/0）"，不给任何"应该怎么调"的指导。但通过让同一工况下的多组标定数据相互对比，车辆**自动学会了**如何优化扭矩响应、能量回收策略等复杂行为。

## 5. GRPO的优缺点

### 优势

1. **无价值网络**：省去一个完整模型的内存和训练成本，是GRPO最大的工程优势。
2. **经验基线更准确**：在价值网络难以训练的场景下，经验基线往往好于一个训练不充分的Critic。
3. **算法简洁**：不需要维护双网络结构和复杂的优势值估计逻辑。
4. **在可验证奖励上表现突出**：数学、代码等有明确正确答案的任务效果极好。
5. **支持多奖励目标**：天然支持多个奖励函数的组合。

### 局限性

1. **组间不一致问题**：组内比较的基线是相对的。同样的回答质量，放在不同组里得到的advantage可能不同——因为基线取决于同组其他回答的质量。
2. **组大小敏感**：G太小信号太差，G太大计算成本过高。
3. **依赖采样多样性**：如果组内所有回答趋同（模式坍塌），学习信号消失。
4. **仅适用于可并行采样的场景**：需要多次生成，推理成本增加。
5. **奖励函数设计关键**：GRPO的效果高度依赖奖励信号的质量。

### 局限性——三电视角

> 组间不一致 = 两轮标定试验，第一轮"好"的阈值可能是80分，第二轮"好"的阈值可能是90分。同一辆车放在不同批次里，得到的评价不同。这不影响批次内的排序，但在跨批次的模型版本对比时需要小心。

## 7. PM视角——三电工程类比

### GRPO vs PPO = 无需基准的对比标定 vs 有标准路谱的标定

| RL概念 | 三电工程类比 |
|--------|------------|
| **PPO的Critic（价值网络）** | 一个"理论最优标定曲线"数据库。标定工程师做任何调整前，先查数据库看"理论上应该是什么值"。但数据库可能不准（欠拟合），也可能太大装不下（内存）。 |
| **GRPO不要Critic** | 标定车辆时，不需要查"理论最优值"，而是在同一工况下让多辆配置相近的车跑一轮，互相比较表现。 |
| **Group（组）** | 一组车（比如5辆）在相同工况（NEDC/WLTC循环、相同温度、相同SOC）下跑出来的数据。 |
| **组内平均奖励 μ_G** | 这组车在该工况下的平均能耗/效率水平。高于平均 = 好，低于平均 = 需要改进。 |
| **组内标准差 σ_G** | 这组车的离散程度。如果5辆车数据非常接近，说明大家都不错或都不行，区分度低。 |
| **KL Penalty** | 标定不能偏离原来的基础标定太远——防止"为优化一个指标把其他指标搞崩了"。 |
| **二元奖励（r=1/0）** | 只看"是否达标"（通过/不通过），不看"超出多少"。虽然信号简单，但足够驱动复杂优化。 |
| **G=2（2-GRPO）** | 只用两台车对比就够——A比B好就够了，不需要5台。大幅降低试验成本。 |

### 一个完整的类比场景

> 标定团队要优化某款电动车的能量回收策略。传统PPO方法需要：
> 1. 建立一个"理论最优能量回收曲线"的数据库（Critic）
> 2. 每次调整策略后，查数据库看"最优值应该是多少"
> 3. 计算调整值和最优值的差距（Advantage）
>
> GRPO的做法是：
> 1. 在同样的WLTC工况下，用8组不同的能量回收参数跑8遍
> 2. 比较8组的电池回馈效率
> 3. 排名前几的参数组合被加强，后几的被抑制
> 4. 不需要知道"理论最优值"——只需要知道"哪组在这个批次里跑得最好"

### 对PM的实操建议

1. **GRPO最适合有明确可验证指标的场景**：数学题（答案对错）、代码（编译通过/测试通过）、三电标定（是否达标）。
2. **算力规划考虑**：GRPO省去了价值模型（可节约约40%显存），但增加了多次采样的推理成本。G=8时推理成本约为单次推理的8倍。
3. **如果价值模型本身质量很高**（有大量标注数据训练过），PPO可能仍然更好。GRPO适合"没时间/没钱训练好Critic"的场景。
4. **2-GRPO是重大工程优化**：如果有算力瓶颈，试试G=2，可能省4-6倍时间而不损失精度。
