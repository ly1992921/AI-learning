## 1. 核心 RL & 对齐公式（PM 版理解）

### 1.1 PPO Clip（近端策略优化裁剪）

```
PPO Clip: L = E[min(r·A, clip(r, 1±ε)·A)]
         r = πθ(新策略) / πold(旧策略)
```

**通俗理解**：模型每次更新时，让新策略和旧策略的"比值"不能差太远——限制在 [1-ε, 1+ε] 范围内（通常ε=0.2，即 ±20%）。

**PM翻译**：就像整车标定时，**每次OT A更新的参数变化量不能超过±20%**。小了进步慢，大了容易崩。

**关键参数 ε=0.2**：这是一个经验值——20%的更新幅度在大多数场景中都是"安全且有效"的。

---

### 1.2 DPO（直接偏好优化）

```
DPO: L = -E[log σ(β·log(πθ(y_w)/πref(y_w)) - β·log(πθ(y_l)/πref(y_l)))]
```

**通俗理解**：让模型更倾向于"人类偏好的回答(y_w)"，远离"人类不喜欢的回答(y_l)"，β控制强度。

**PM翻译**：相当于**A/B测试**——把好方案（A）和差方案（B）同时给模型看，告诉它"A更好，以后多学A的样子"。

**关键参数 β**：β越大，模型对偏好的"记忆"越强（但可能过拟合）；β越小，模型越灵活但可能学不到偏好。

---

### 1.3 GRPO（组相对策略优化）

```
GRPO: Â = (r_i - μ_G) / σ_G
      然后做PPO clip更新（不需要价值模型Critic）
```

**通俗理解**：在一组生成结果中，看单个结果比组的平均值好多少/差多少，据此调整。

**PM翻译**：不是用外部标准评判，而是**同一批产品内部比较**——A方案比这批平均好20%，B方案差10%，对A奖励、对B惩罚。

**核心优势**：不需要训练"评分员"（价值模型），DeepSeek-R1的核心技术。

---

### 1.4 KTO（Kahneman-Tversky 优化）

```
KTO: L = λ_w·(1-v(y_w)) + λ_l·v(y_l)
     v = σ(β·log(πθ/πref) - z)
```

**通俗理解**：不需要成对的"好vs坏"例子，只要有单独的好回答或坏回答就能训练。

**PM翻译**：DPO需要A/B对比数据（两个答案哪个好），KTO只需要**单独的"好评"或"差评"**——数据收集成本更低。

---

### 1.5 IPO（身份偏好优化）

```
IPO: L = E[(log(πθ(y_w)/πref(y_w)) - log(πθ(y_l)/πref(y_l)) - 1/(2β))²]
```

**通俗理解**：直接优化"好回答和坏回答的对数概率差距"，而不是用PPO的裁剪方法。

**PM翻译**：把"A比B好多少"变成一个固定的优化目标，训练更稳定。

---

### 1.6 GAE（广义优势估计）

```
GAE: Â_t = Σ(γλ)^l · δ_{t+l}
     δ_t = r_t + γV(s_{t+1}) - V(s_t)
```

**通俗理解**：不是只看当前这一步的"好坏"，而是综合考虑未来多步的回报，γ平衡"短期vs长期"。

**PM翻译**：评价一个策略决策时，**不只盯着当下的效果，还要考虑后续影响**——就像标定电池策略时，不光看当前SOC精度，还要看未来10分钟的预测效果。

**关键参数**：
- **γ（折扣因子）** ≈ 0.99：长期眼光（越接近1越看长远）
- **λ（GAE参数）** ≈ 0.95：平滑程度

---

### 1.7 KL 惩罚（KL Penalty）

```
R_total = r(x,y) - β · KL(πθ ∥ πref)
```

**通俗理解**：在追求高分的同时，限制模型不能偏离原始能力太远。

**PM翻译**：**"既要又要"的平衡**——既要模型越来越听话（分数高），又不能让它忘了最基本的知识（偏离原始模型）。β越大越保守。

---

### 1.8 奖励模型（Bradley-Terry）

```
L = -E[log σ(r(x,y_w) - r(x,y_l))]
```

**通俗理解**：奖励模型通过对比"好回答和差回答"来学习打分——A比B好→给A高分。

---

### 1.9 Best-of-N

```
y* = argmax r(y_i), y_i ~ πθ, i=1..N
```

**通俗理解**：让模型生成N个答案，选得分最高的那个。N越大，质量越高但成本也越高。

**PM翻译**：**"不合格就多试几次"**——面试官可以多面试几个候选人，选最好的那个。N=1最便宜，N=64质量最高但贵64倍。

## 2. Transformer & 架构公式（PM 版）

### 2.1 Self-Attention（自注意力）

```
Attn(Q,K,V) = softmax(QKᵀ/√d_k) · V
```

**通俗理解**：每个词通过Q（我关心的）去匹配所有词的K（它们有什么），根据匹配度加权组合V（它们的内容）。

**PM翻译**：**每个词都在做一次"信息检索"**——搜索全句，找出最相关的内容加权吸收。

---

### 2.2 Multi-Head Attention（多头注意力）

```
MHA(X) = Concat(head₁,...,head_h) · W_O
head_i = Attn(XW_Q, XW_K, XW_V)
```

**通俗理解**：不是只看一个方面，而是并行看多个方面——语法关系、语义关系、位置关系各用一个"头"。

**参数**：常用32个或8个头（GQA方案）。

**PM翻译**：**多个视角同时看问题**——产品评审时，同时有技术、市场、用户、成本的多个专家视角。

---

### 2.3 RoPE（旋转位置编码）

```
f(x_m, m) = x_m · e^{imθ_j}, θ_j = 10000^{-2j/d}
```

**通俗理解**：用"旋转操作"给每个词标位置。位置越远的两个词，向量之间"旋转角度"越大。

**核心特性**：两个词的注意力权重只取决于它们的**相对位置**，不是绝对位置。

---

### 2.4 LoRA（低秩适配）

```
W' = W + (α/r) · BA
B ∈ R^{d×r}, A ∈ R^{r×k}
```

**关键参数**：
- **r（秩）**：自由度，越大越接近全量微调（8-64范围）
- **α（alpha）**：更新幅度，越大更新越猛

**PM翻译**：不改全部参数，只改一个小矩阵。**r=8是最省钱的方案，r=64效果接近全量微调但更贵。**

---

### 2.5 知识蒸馏

```
L_KD = (1-α)·L_CE(y, ŷ) + α·T²·KL(p_teacher ∥ p_student)
```

**通俗理解**：让"大模型（老师）"教"小模型（学生）"——学生模仿老师的输出分布，学老师的"思路"而不是死记硬背答案。

**关键参数**：
- **α**：老师的影响程度（0=全靠标准答案，1=全靠模仿老师）
- **T（温度）**：蒸馏的"细腻程度"

**PM翻译**：**资深工程师带新人**——新人不是自己从头摸索，而是模仿资深工程师的思考方式。

---

### 2.6 SwiGLU（FFN 激活函数）

```
FFN(x) = (Swish(xW₁) ⊙ xW₃) · W₂
```

**通俗理解**：比普通FFN多了额外一步"门控"操作——决定哪些信息要保留、哪些要丢弃。

**实际效果**（原书）：SwiGLU在同等参数量下比ReLU好约2-3%。

## 3. 解码方法速查

| 方法 | 公式/规则 | 关键参数 | PM理解 |
|------|----------|---------|--------|
| **Greedy** | 每次选概率最高的词 | — | 贪心算法——每次都选最稳妥的，但可能错过更好的整体方案 |
| **Beam Search** | 保持Top-B条候选路径 | B=4-8 | 广撒网——同时探索B条"可能"路径，选整体最优的 |
| **Temperature** | 调整概率分布的"陡峭度" | T∈[0.1, 1.5] | T低=保守/确定性（像稳妥驾驶），T高=有创意/随机性（像探索性驾驶） |
| **Top-k** | 只看概率最高的k个词 | k=40-100 | 剪掉明显不好的选择，只在"候选名单"里选 |
| **Top-p (Nucleus)** | 保留概率加起来达到p的最小集合 | p=0.9-0.95 | 动态剪枝——好词多就多留几个，好词少就少留 |
| **Min-p** | 保留概率≥p·P_max的词 | p=0.05-0.1 | 相对阈值——比最高概率低太多的都不要 |
| **Repetition Penalty** | 对已经出现过的词降分 | θ=1.1-1.3 | 防止车轱辘话——说过的话再说就被扣分 |

**PM决策指南**：
- **创意写作（营销文案）**：高温+Top-p → 有创意
- **事实回答（产品文档）**：低温+Top-k → 准确
- **代码生成**：Beam Search → 稳妥

## 4. 系统 & 并行计算速查

### 4.1 模型内存估算

| 公式 | 70B模型估算 | 说明 |
|------|------------|------|
| **权重存储** = 2P | 140 GB (BF16) | 模型参数占的基本空间 |
| **Adam优化器** = 2P×4 | 280 GB | 优化器状态（动量和方差） |
| **全训练占用** ≈ 8P | 560 GB | 权重+优化器+梯度 |
| **FSDP分片/GPU** = 8P/N | 70 GB (8GPU) | 分布式训练后每GPU的负担 |
| **生成时算力强度** = 2P/2P = 1 | 1 FLOP/Byte | 推理是严重的内存瓶颈型 |
| **Token生成速率** = HBM_BW/(2P) | ~14 tok/s (A100) | 每秒生成约14个token |

**PM启示**：
- 70B模型**训练需要至少8张H100（共640GB显存）**
- LoRA微调只需要约48GB（一张A100就够了）
- 推理时的瓶颈不是计算速度，是**内存带宽**

### 4.2 分布式训练通信量

| 并行方式 | 每层通信量 | 说明 |
|---------|-----------|------|
| **TP（张量并行）** | ~188 MB (70B, TP=8) | 每个transformer层内部通信 |
| **PP（流水线并行）** | 气泡率 = (P-1)/(P+M-1) | P=流水线阶段数 |

**PM启示**：分布式训练时，GPU之间的通信速度（NVLink/InfiniBand）决定了加速效率——不是GPU越多越好，通信瓶颈会限制扩展。

## 6. 关键超参数范围

| 参数 | 典型范围 | PM理解 |
|------|---------|--------|
| **学习率（LR）** | 1e-4 到 3e-4 | 调整步长——太大学蹦，太小永远到不了 |
| **Batch Size** | 数百万tokens（预训练） | 并行度——越大GPU利用率越高，但显存要求也高 |
| **LoRA秩(r)** | 8-64 | 定制力度——8最省、64最接近全量 |
| **LoRA α** | 16-32 | 更新幅度 |
| **PPO clip ε** | 0.2 | 每次更新的最大偏离（±20%） |
| **KL β** | 0.01-0.1 | 保守程度——越大越保守 |
| **温度 T** | 0.1-1.5 | 创造性——低=保守，高=创意 |
| **Top-p** | 0.9-0.95 | 候选集覆盖率 |
| **Top-k** | 40-100 | 候选词数量 |
| **梯度裁剪阈值** | 1.0 | 安全护栏——超过1的梯度被限流 |
| **蒸馏温度 T** | 2-4 | 老师指导的细腻程度 |
| **GAE λ** | 0.95 | 优势估计的平滑度 |

## 7. Agent 设计模式决策树

原书给出的选择指南：

```
需要什么类型的Agent?
├── 简单、确定性的工作流（先A再B再C）
│   └── Prompt Chaining（提示链）
├── 输入需要分类后分发给不同模块
│   └── Routing（路由）
├── 多路独立处理再合并结果
│   └── Parallelization（并行化）
├── 一个主Agent调度多个子Agent
│   └── Orchestrator-Workers（编排器-工作者）
├── 需要不断优化结果（循环改进）
│   └── Evaluator-Optimizer（评估器-优化器）
└── 复杂、自主的任务
    └── ReAct / Planning Agents（自主智能体）
```

## 8. 常见失败模式及修复

| 失败模式 | 症状 | 可能原因 | 修复方向 |
|---------|------|---------|---------|
| **Loss 发散** | 训练损失不断上升 | 学习率太高 | 降低LR或加梯度裁剪 |
| **模式崩溃** | 模型总是输出同样的内容 | KL惩罚太小 | 增大β值 |
| **Reward Hacking** | 分数高但质量差 | 奖励信号设计有漏洞 | 重新设计奖励函数 |
| **过拟合** | 训练分数高，测试分数低 | 数据太少/LoRA秩太高 | 加正则化/降秩 |
| **梯度爆炸** | 训练不稳定 | 梯度太大 | 加梯度裁剪 |
| **注意力稀释** | 长上下文表现差 | 中间信息被忽略 | 重新排列关键信息位置 |
