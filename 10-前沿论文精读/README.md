# 10 - 前沿论文精读

> 持续跟踪前沿是AI PM的核心能力。本章提供论文精读的方法论和模板。

## 📚 精读方法论

### 三遍阅读法

1. **第一遍（15分钟）**: 标题 → 摘要 → 结论 → 图表
   - 判断：这篇论文值不值得精读？
   - 产出：一句话总结

2. **第二遍（1小时）**: 逐节阅读，关注核心贡献
   - 理解：问题是什么？方法是什么？结果如何？
   - 产出：结构化笔记

3. **第三遍（2-3小时）**: 深入数学推导和实验细节
   - 复现：关键实验结果
   - 产出：深度分析笔记

### 论文评估框架

| 维度 | 问题 | 评分 |
|------|------|------|
| 问题重要性 | 解决的是核心问题吗？ | 1-5 |
| 方法创新性 | 方法有显著创新吗？ | 1-5 |
| 实验充分性 | 实验设计充分吗？ | 1-5 |
| 可扩展性 | 方法能否扩展到更大规模？ | 1-5 |
| 产品相关度 | 对产品工作有直接启发吗？ | 1-5 |

## 📝 精读笔记模板

```markdown
# [论文标题]

**作者**: [作者列表]
**年份/会议**: [年份] [会议]
**链接**: [arXiv/IEEE等链接]
**精读日期**: YYYY-MM-DD
**推荐等级**: A / B / C

---

## 一句话总结
[用一句话概括论文的核心贡献]

## 研究背景
[这个问题为什么重要？前人做了什么？]

## 核心方法
[论文提出的方法是什么？关键创新点？]

## 实验结果
[主要实验结论，最好有具体数字]

## 关键图表
[描述最重要的1-2个图/表]

## PM视角分析
### 产品机会
[这项技术可能催生什么产品？]

### 技术风险
[落地的主要挑战是什么？]

### 竞品关联
[哪些公司/产品在做类似的事？]

## 待深入问题
- [ ] 问题1
- [ ] 问题2

## 相关论文
- [论文1](链接) — 关系说明
- [论文2](链接) — 关系说明
```
